Basic GIT commands for this class
Getting a local copy of the repository
This GIT command will create a local copy of the remote code repository.
git clone https://github.com/em1082/mysite.git
Listing all available branches
GIT allows you to create branches where you can experiment without affecting your original files. I will be creating a new branch containing each class' code. This structure will let you compare differences between classes simply by checking out the branches you want to compare. To see all of the branches available in the remote repository, run the following command.
git branch -r
To get a copy of a class branch
At the end of each classs I will "push" my changes to the remote repository at GitHub. To refresh your local copy of our repository containing all of our changes you'll use these commands
Change your local repository to point to the master branch:
git checkout master
Download changes from GitHub and merge them into your copy of the repository:
git pull
List all of the available branches now in your local copy:
git branch
Switch over to one of the branches:
git checkout BRANCH_NAME
Creating your own sandbox branches
One of the best ways to learn to code is to play with it. Git lets you create your own branches to test out your ideas. When you're done you can just delete your branch and none of your original files will have been changed. To create a sandbox branch:
Start by checking out the branch you would like as your starting point:
git checkout 02_VARIABLES
Then dream up a name for your new branch and type:
git checkout -b my_new_branch
You can now edit your copy of the files as much as you want and can always get back to the original.
